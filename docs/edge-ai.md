# Edge AI компонент

## 1. Ціль edge-моделі

На edge-рівні система повинна класифікувати поточний стан повітря в одну з чотирьох категорій:

- `добре`
- `помірне`
- `шкідливе`
- `небезпечне`

Локальна класифікація дозволяє:

- спрацьовувати без зв'язку з хмарою
- зменшити затримку реагування
- відсікати частину простих правил ще на gateway або пристрої

## 2. Вибір моделі

Пропонується компактна `int8` `MLP`-модель з двома прихованими шарами.

Параметри:

- вхід: `12` ознак
- шари: `12 -> 16 -> 8 -> 4`
- квантування: `int8`
- орієнтовний розмір: `30-60 KB`
- очікуваний inference:
  - `15-30 ms` на `ARM Cortex-M7`
  - `< 10 ms` на `Raspberry Pi`

Таке рішення виконує обмеження:

- розмір `< 1 MB`
- inference `< 100 ms`

## 4. Автономна логіка на пристрої

Після кожного нового вимірювання edge-вузол:

1. нормалізує значення
2. рахує коротке rolling window
3. виконує inference
4. запускає локальне правило реагування

Правила реагування:

- `добре` -> зелений LED
- `помірне` -> жовтий LED
- `шкідливе` -> червоний LED + короткий buzzer
- `небезпечне` -> червоний LED + повторний buzzer + локальний журнал події

## 5. Поведінка при втраті зв'язку

- останні `N` вимірювань буферизуються локально
- локальна класифікація продовжується без cloud
- якщо клас `небезпечне` триває довше `3` інтервалів, gateway піднімає прапорець аварійного стану
- після відновлення каналу буфер дозавантажується в хмару

## 6. Оновлення моделі

Основна стратегія: `periodic retraining`.

Життєвий цикл:

- раз на тиждень збираються анонімізовані агрегати з `InfluxDB`/`PostgreSQL`
- в хмарі виконується перенавчання
- модель проходить `offline` перевірку на контрольному датасеті
- при успішних метриках модель пакується в OTA-оновлення
- оновлення поширюється канареєчно: `10% -> 50% -> 100%`

Додатково можна розглянути `federated learning`, якщо в майбутньому з'явиться потреба оновлювати модель без передавання великого обсягу сирих даних.

## 7. Метрики якості моделі

- `macro F1` по 4 класах
- `false negative rate` для класу `небезпечне`
- середній час inference
- частка рішень, що збігаються з cloud-аналітикою

## 8. Псевдокод пристрою

```text
read_sensors()
update_rolling_window()
features = build_feature_vector()
label = classify(features)

if label == "dangerous":
    led = RED
    buzzer = ON
elif label == "harmful":
    led = RED
    buzzer = SHORT_BEEP
elif label == "moderate":
    led = YELLOW
else:
    led = GREEN

buffer_measurement()
publish_if_link_available()
```
